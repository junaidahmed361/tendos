# Tendos v0.1.1 (Draft Release Notes)

## Highlights

This patch release improves governance/docs accuracy and adds harness declaration support for cartridge policies.

## Added

- Harness declaration support in cartridge manifests via `harness`:
  - `security_guardrails`
  - `pii_redaction`
  - `update_sync`
- Loader validation for `harness.yaml_path` file existence
- `tendos init` now scaffolds `harness/harness.yaml`
- Dogfood local cartridge example:
  - `examples/hermes-local-agent/`

## Changed

- Removed non-existent project email addresses from docs (`SECURITY.md`, `CODE_OF_CONDUCT.md`)
- Security reporting now points to GitHub private vulnerability reporting flow
- README updated to include Nintendo + Docker naming origins and harness YAML declarations

## Build Artifacts

- `dist/tendos-0.1.1.tar.gz`
- `dist/tendos-0.1.1-py3-none-any.whl`

## Upgrade

```bash
pip install --upgrade tendos==0.1.1
# or
uv add tendos==0.1.1
```
